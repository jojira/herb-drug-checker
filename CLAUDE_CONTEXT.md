## Project Summary

**Brand:** Formulens | **Domain:** formulens.co  
A Next.js (App Router) web application for licensed acupuncturists and TCM practitioners to
cross-reference Western pharmaceuticals with TCM herbs and formulas. Surfaces herb-drug
interactions at the point of care. Deployed on Vercel.

**Repo path:** `C:\Users\josep\projects\herb-drug-checker`  
**Stack:** Next.js + TypeScript (strict) + Tailwind CSS + Clerk auth + Vercel

## Hard Rules (Never Break These)

1. **Never import `lib/interactionEngine.ts` in a client component** — server-only, imports JSON
2. **All shared types from `lib/types/clinical.ts`** — never redefine `TrustTier`, `SeverityLevel`, `InteractionEngineResult`
3. **`checkInteractions()` is async** (v4.0+) — all callers must `await` it
4. **Disclaimer is non-negotiable** — on every page and every result, never dismissible, never hidden
5. **Fuse.js runs server-side only** — never move search to client components
6. **NCCAOM naming** — every Gold herb needs: `pinyin`, `latin`, `english`, `nccaom_code`
7. **Severity escalation** — worst-case always wins, never downgrade
8. **mockInteractions.json has `data_status: "mock_unverified"`** — never remove until pharmacist review
9. **Clerk components** — use `<Show when="signed-in">` NOT deprecated `<SignedIn>` / `<SignedOut>`
10. **No inline styles** — Tailwind classes only

## Auth / Search Limit

- **Guests:** 5 completed interaction checks (stored in `localStorage["formulens_search_count"]`)
- **Authenticated:** Unlimited
- Counter increments after successful result only, never resets automatically
- `SoftWallModal` appears when guest limit is reached

## Common Gotchas (Historical Bugs)

| Bug | Correct approach |
|-----|-----------------|
| JS single quotes in onclick attrs inside Python f-strings | Use double quotes: `onclick="fn()"` |
| Bare `\n` in JS string literals inside Python strings | Use `\\n` (double escape) |
| `JSON.stringify()` inside single-quoted JS attr | Store in array, pass index: `onclick="fnByIdx(i)"` |
| Importing `interactionEngine.ts` in client component | Use API route instead: `POST /api/interactions` |
| Redefining `TrustTier` in individual files | Import from `lib/types/clinical.ts` |
